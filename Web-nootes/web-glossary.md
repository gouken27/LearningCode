## Определения

### a11y
**Accessibility** — доступность. Число 11 = количество букв между «a» и «y» в слове «accessibility». Практика проектирования интерфейсов так, чтобы ими могли пользоваться люди с любыми ограничениями: незрячие (скринридеры), слабовидящие, пользователи только с клавиатурой, глухие и т.д.  
→ *Стандарт: [WCAG](#wcag). Инструмент браузера: вкладка Accessibility в DevTools.*

---

### API
**Application Programming Interface** — интерфейс для взаимодействия программ между собой. Набор правил: «если пришлёшь такой запрос — получишь такой ответ». В веб-разработке обычно подразумевает HTTP-API сервера.  
→ *Архитектурный стиль: [REST](#rest). Альтернатива: GraphQL.*

---

### AST
**Abstract Syntax Tree** — абстрактное синтаксическое дерево. Структурированное представление исходного кода, которое строит парсер. `const x = 2 + 3` превращается в дерево: `VariableDeclaration → BinaryExpression → Literal(2) + Literal(3)`. Используется компиляторами, транспиляторами (Babel), линтерами (ESLint), форматтерами (Prettier).

---

### CORS
**Cross-Origin Resource Sharing** — механизм безопасности браузера. Запрещает JavaScript делать запросы к другому домену/порту без явного разрешения сервера через заголовок `Access-Control-Allow-Origin`. Ограничивает только браузер — `curl` и Postman не затрагивает.  
→ *Настраивается на сервере: `app.use(cors({ origin: 'https://mysite.com' }))`.*

---

### CLS
**Cumulative Layout Shift** — совокупный сдвиг макета. Метрика Core Web Vitals: насколько элементы страницы «прыгают» во время загрузки. Высокий CLS = плохой UX и хуже SEO. Частая причина — картинки без `width`/`height` атрибутов.

---

### CRUD
**Create, Read, Update, Delete** — четыре базовые операции с данными. Маппинг на HTTP: `POST` → Create, `GET` → Read, `PUT/PATCH` → Update, `DELETE` → Delete.

---

### CSR
**Client-Side Rendering** — рендеринг на стороне клиента. Сервер отдаёт пустой `<div id="root">`, JavaScript скачивается и рисует всю страницу в браузере. Подход React/Vue/Angular по умолчанию. Плюсы: отзывчивый UI. Минусы: медленная первая загрузка, плохое SEO.  
→ *Альтернативы: [SSR](#ssr), [SSG](#ssg).*

---

### CSSOM
**CSS Object Model** — аналог DOM для CSS. Браузер строит CSSOM из CSS-файлов параллельно с DOM. Render Tree = DOM + CSSOM. Без CSSOM браузер не может рисовать страницу — CSS блокирует рендеринг.

---

### CSS
**Cascading Style Sheets** — каскадные таблицы стилей. Язык описания внешнего вида HTML-элементов. «Каскадные» — потому что стили применяются по правилам приоритета (каскаду): специфичность, порядок, источник.

---

### CLI
**Command Line Interface** — интерфейс командной строки. Управление программой через текстовые команды в терминале. Противоположность GUI. В веб-разработке: `npm`, `git`, `node`, `vite` — всё CLI.

---

### DNS
**Domain Name System** — система доменных имён. «Телефонная книга» интернета: переводит доменное имя (`anime-catalog.dev`) в IP-адрес (`104.21.64.0`). DNS-сервера распределены по всему миру иерархически.

---

### DOM
**Document Object Model** — объектная модель документа. Браузер строит DOM из HTML: дерево объектов, где каждый тег — узел. JavaScript работает с DOM через `document.querySelector`, `createElement` и т.д. DOM ≠ HTML-файл: JS может изменить DOM без изменения исходного файла.

---

### DRY
**Don't Repeat Yourself** — «не повторяй себя». Принцип программирования: каждый фрагмент знания должен иметь единственное, однозначное представление в системе. Нарушение DRY = копипаст, который потом нужно менять в N местах.

---

### ESM
**ES Modules** — стандартная модульная система JavaScript (ES2015+). `import`/`export`. Статические (известны при парсинге) — поддерживают tree-shaking. Альтернатива: [CommonJS](#cjs) (`require`/`module.exports`) — Node.js legacy.  
→ *В браузере: `<script type="module">`. В Node: `"type": "module"` в package.json.*

---

### FCP
**First Contentful Paint** — первая отрисовка контента. Метрика Core Web Vitals: время от начала загрузки до появления первого текста или картинки на экране. Хорошо: < 1.8с.

---

### FOUC
**Flash of Unstyled Content** — мелькание не стилизованного контента. Момент когда HTML уже загружен, а CSS ещё нет — страница показывается без стилей на долю секунды. Решение: `<link rel="stylesheet">` в `<head>`.

---

### HTML
**HyperText Markup Language** — язык разметки гипертекста. Описывает структуру и смысл контента через теги. Не язык программирования (нет циклов, переменных, условий). «HyperText» = текст со ссылками на другие тексты.

---

### HTTP
**HyperText Transfer Protocol** — протокол передачи гипертекста. Текстовый протокол запрос-ответ для обмена данными в вебе. Клиент отправляет запрос (метод + URL + заголовки + тело), сервер отвечает (статус + заголовки + тело). Без шифрования — данные передаются открыто.

---

### HTTPS
**HTTP Secure** = HTTP + TLS-шифрование. Все данные между браузером и сервером зашифрованы. Браузеры помечают HTTP-сайты как «небезопасные». Обязателен для: паролей, платёжных данных, любых персональных данных, PWA.

---

### IDE
**Integrated Development Environment** — интегрированная среда разработки. Редактор кода + встроенные инструменты: автодополнение, дебаггер, рефакторинг, Git-интеграция. Примеры: VS Code, WebStorm, IntelliJ.

---

### IIFE
**Immediately Invoked Function Expression** — немедленно вызываемое функциональное выражение. `(function() { ... })()` — создаётся и тут же вызывается. До ES Modules использовался для изоляции области видимости. Сейчас устарел — замените на модули.

---

### JIT
**Just-In-Time compilation** — компиляция «на лету». Код интерпретируется, но «горячие» (часто выполняемые) участки компилируются в машинный код прямо во время работы программы. Используется в V8 (JavaScript), JVM (Java), .NET (C#). Сочетает плюсы интерпретации (не нужна ручная сборка) и компиляции (скорость).

---

### JS
**JavaScript** — язык программирования для веба. Изначально — только браузерный скриптинг. Сейчас — универсальный: браузер, сервер (Node.js), мобильные приложения (React Native), десктоп (Electron). Динамически типизирован. Использует JIT-компиляцию (V8).

---

### JSON
**JavaScript Object Notation** — текстовый формат обмена данными. Выглядит как JS-объект, но является строкой с правилами: только двойные кавычки, нет комментариев, нет `undefined`, нет функций. Стандарт де-факто для REST API.  
→ *Парсинг: `JSON.parse(str)`. Сериализация: `JSON.stringify(obj)`.*

---

### JWT
**JSON Web Token** — компактный самодостаточный токен аутентификации. Состоит из трёх частей (разделены точкой): header (алгоритм), payload (данные пользователя), signature (подпись). Сервер проверяет подпись — не нужно обращаться к БД при каждом запросе.  
→ *Хранить: `localStorage` (уязвим к XSS) или `httpOnly cookie` (безопаснее).*

---

### LCP
**Largest Contentful Paint** — отрисовка наибольшего контента. Метрика Core Web Vitals: время до отображения самого большого видимого элемента (обычно hero-изображение или заголовок). Хорошо: < 2.5с.

---

### LTS
**Long Term Support** — версия с долгосрочной поддержкой. В Node.js — чётные версии (18, 20, 22), поддерживаются 3 года. Для продакшена и обучения — всегда LTS.

---

### MPA
**Multi-Page Application** — традиционное веб-приложение: каждый переход = новый HTML-документ с сервера. Хорошее SEO, простая архитектура. Противоположность [SPA](#spa).

---

### npm
**Node Package Manager** — менеджер пакетов для Node.js. Два в одном: CLI-инструмент (`npm install`) и реестр пакетов (npmjs.com, 2+ млн пакетов). Альтернативы: `pnpm` (быстрее), `yarn` (популярен в корпоративной среде).

---

### ORM
**Object-Relational Mapping** — объектно-реляционное отображение. Библиотека, которая позволяет работать с БД через объекты JS/Python/Java вместо написания SQL-запросов вручную. Примеры для Node.js: Prisma, Drizzle, Sequelize, TypeORM.

---

### PAT
**Personal Access Token** — персональный токен доступа. Строка-заменитель пароля для работы с API (GitHub, GitLab и т.д.) по HTTPS. Более безопасен, чем пароль: можно ограничить права и отозвать в любой момент.

---

### REST
**Representational State Transfer** — не протокол, а архитектурный стиль API. Принципы: клиент-сервер, stateless (каждый запрос самодостаточен), кешируемость, единый интерфейс (URL = ресурс, метод = действие). API, следующий этим принципам, называют RESTful.

---

### REPL
**Read-Eval-Print Loop** — интерактивная консоль: читает ввод, выполняет, печатает результат, повторяет. `node` без аргументов запускает Node REPL. Браузерный DevTools Console — тоже REPL.

---

### SPA
**Single Page Application** — одностраничное приложение. Весь UI строится JavaScript в браузере; переходы между «страницами» — смена контента без перезагрузки. Примеры: React, Vue, Angular приложения. Плюсы: плавный UX. Минусы: плохое SEO без SSR, медленная первая загрузка.

---

### SQL
**Structured Query Language** — язык структурированных запросов. Стандартный язык для работы с реляционными БД: SELECT, INSERT, UPDATE, DELETE, JOIN. Диалекты: SQLite, PostgreSQL, MySQL — все поддерживают базовый SQL, но отличаются в деталях.

---

### SSG
**Static Site Generation** — генерация статичных страниц при сборке (не при запросе). Markdown → HTML при `npm run build`. Очень быстро (нет сервера), отлично для SEO. Инструменты: Astro, Eleventy, Hugo. Ограничение: контент обновляется только при пересборке.

---

### SSR
**Server-Side Rendering** — рендеринг на сервере. HTML генерируется на сервере при каждом запросе и отправляется браузеру готовым. Хорошее SEO, быстрая первая загрузка. Инструменты: Next.js (React), Nuxt (Vue). Требует Node.js сервер.

---

### TDZ
**Temporal Dead Zone** — «временная мёртвая зона». Период между началом блока `{}` и объявлением `let`/`const`, когда переменная существует но недоступна. Обращение в TDZ = `ReferenceError`. Это намеренная фича для предотвращения ошибок с `hoisting`.

---

### UX
**User Experience** — пользовательский опыт. Совокупность ощущений пользователя при взаимодействии с продуктом. Включает: юзабилити, доступность, производительность, визуальный дизайн. UX-дизайнер исследует поведение пользователей и проектирует интерфейсы.

---

### UI
**User Interface** — пользовательский интерфейс. Визуальная часть: кнопки, формы, цвета, типографика, иконки. UI-дизайнер создаёт макеты. Фронтенд-разработчик реализует их в коде.

---

### URL
**Uniform Resource Locator** — унифицированный указатель ресурса. Стандартный адрес ресурса в интернете.  
```
https://anime-catalog.dev/api/franchises?status=ongoing#results
─────   ─────────────────  ─────────────────  ───────────  ───────
схема   хост               путь               query-строка  якорь
```

---

### VCS
**Version Control System** — система контроля версий. Отслеживает историю изменений файлов, позволяет откатиться, работать параллельно в ветках. Примеры: Git (DVCS), SVN (CVCS).  
→ *Подробный курс: [git-guide.md](../git-guide.md).*

---

### WCAG
**Web Content Accessibility Guidelines** — руководство по доступности веб-контента. Стандарт W3C. Уровни: A (минимум), AA (стандарт), AAA (максимум). Уровень AA требуется по закону во многих странах. Минимальный контраст текста по AA: 4.5:1.

---

### XSS
**Cross-Site Scripting** — межсайтовый скриптинг. Уязвимость: злоумышленник внедряет вредоносный JavaScript в страницу через пользовательский ввод. Если сайт вставляет данные пользователя через `innerHTML` без экранирования — XSS возможен.  
→ *Защита: `textContent` вместо `innerHTML`, Content-Security-Policy заголовок, DOMPurify.*

## Термины (алфавитный порядок)

| Термин | Определение |
|---|---|
| **Babel** | Транспилятор JS. Преобразует современный ES2024+ в ES5 для старых браузеров. Читает AST, трансформирует, генерирует код. |
| **bcrypt** | Алгоритм хеширования паролей. Намеренно медленный (настраивается «стоимость»), с солью — защита от радужных таблиц. |
| **BEM** | Block-Element-Modifier. Методология именования CSS-классов: `.card`, `.card__title`, `.card--featured`. |
| **Бандлер** | Инструмент сборки. Берёт много файлов с импортами, объединяет в один (или несколько). Примеры: Vite, Webpack, Rollup. |
| **Браузер** | Программа для отображения веб-страниц. Основные движки: Blink (Chrome, Edge), Gecko (Firefox), WebKit (Safari). |
| **Брейкпоинт** | Точка в CSS, где меняется layout при определённой ширине (`@media (min-width: 768px)`). |
| **Вебхук** | HTTP-коллбек: сервис отправляет POST-запрос на ваш URL при событии (пуш в GitHub → сборка на Netlify). |
| **Виртуальный DOM** | Лёгкое JS-представление DOM в памяти. React сравнивает старый и новый Virtual DOM, применяет минимальные изменения к реальному DOM. |
| **Дедлайн ошибки** | Ошибка `throw` только при исключительной ситуации, не для управления потоком. |
| **Деплой** | Публикация приложения: загрузить код на сервер так, чтобы он стал доступен пользователям через интернет. |
| **Деструктуризация** | Синтаксис ES6: `const { title, rating } = franchise`. Извлечь значения из объекта или массива в переменные. |
| **Делегирование событий** | Паттерн: один обработчик на родителе вместо N обработчиков на детях. Работает с динамически созданными элементами. |
| **Замыкание** | Функция + её лексическое окружение. Функция «помнит» переменные внешней функции даже после её завершения. |
| **Индекс (БД)** | Структура данных для ускорения поиска. Как оглавление книги: не перебирать все строки, а перейти сразу. Замедляет запись. |
| **Каскад (CSS)** | Алгоритм разрешения конфликтов стилей: источник → специфичность → порядок. |
| **Компилятор** | Программа, переводящая код из одного языка в другой (обычно в машинный код или промежуточное представление). |
| **Контейнер (Docker)** | Изолированная среда запуска приложения со всеми зависимостями. «Работает у меня» → «работает везде». |
| **Линтер** | Инструмент статического анализа кода. Находит ошибки и нарушения стиля без запуска. ESLint для JS, Stylelint для CSS. |
| **Маршрутизатор (роутер)** | Код, определяющий какой обработчик вызвать для данного URL. В Express: `app.get('/api/franchises', handler)`. |
| **Метод HTTP** | Семантика операции: GET (читать), POST (создать), PUT (заменить), PATCH (обновить), DELETE (удалить). |
| **Middleware** | «Промежуточный слой» в Express. Функция `(req, res, next)`, стоящая между запросом и обработчиком. CORS, логирование, авторизация. |
| **Минификация** | Удаление пробелов, переносов, комментариев из кода без изменения функциональности. Уменьшает размер файлов. |
| **Нормализация (БД)** | Разбивка данных на связанные таблицы для устранения дублирования. Вместо хранить жанры в каждой записи — отдельная таблица genres. |
| **Пагинация** | Разбивка большого списка данных на страницы. `LIMIT 20 OFFSET 40` → третья страница по 20 записей. |
| **Парсер** | Программа, читающая текст по правилам грамматики и строящая структурированное представление. HTML-парсер → DOM. |
| **Полифилл** | Код, реализующий стандартную функцию в браузерах где её нет. `Array.prototype.at` → полифилл для IE. |
| **Пропп (props)** | В React: данные передаваемые в компонент снаружи. Аналог атрибутов HTML. |
| **Реактивность** | Автоматическое обновление UI при изменении данных. `useState` в React, `ref`/`reactive` в Vue. |
| **Рефактор** | Улучшение структуры кода без изменения функциональности. |
| **Семантика (HTML)** | Использование тегов по их назначению: `<nav>` для навигации, `<article>` для контента, `<button>` для действий. |
| **Сессия** | Механизм хранения состояния между запросами на сервере. ID сессии хранится в куках, данные — в БД или памяти. |
| **Спецификация** | Официальный стандарт поведения: ECMAScript (JS), HTML Living Standard, CSS Specifications. |
| **Специфичность** | Вес CSS-селектора. Определяет какое из конфликтующих правил победит. `id > class > tag`. |
| **Стейт (state)** | Состояние. Данные, от которых зависит текущий вид UI: открыта ли модалка, что выбрано в фильтре, список загружен. |
| **Транзакция (БД)** | Набор операций, выполняемых атомарно: либо все, либо ни одной. Защита от частичных обновлений при сбое. |
| **Транспилятор** | Компилятор из одного языка/версии в другой. TypeScript → JS, JSX → JS, SCSS → CSS. |
| **Tree shaking** | Удаление неиспользуемого кода при сборке. Работает только с ESM (статические импорты). |
| **Хеширование** | Преобразование данных в строку фиксированной длины. Одностороннее: нельзя восстановить исходное. SHA-256, bcrypt. |
| **Хостинг** | Сервис хранения и обслуживания файлов сайта. Статический: Netlify, Vercel. Серверный: Railway, Render, DigitalOcean. |
| **Хук (hook)** | В React: функция `use*` для добавления состояния и эффектов в функциональный компонент (`useState`, `useEffect`). |
| **Эндпоинт** | Конкретный URL API: `/api/franchises`, `/api/franchises/:id`. Точка входа для одного типа запросов. |
